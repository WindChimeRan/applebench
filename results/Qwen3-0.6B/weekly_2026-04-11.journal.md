### inferrs
- **Error**: `500, message='Internal Server Error'` on all 300 requests (100 x 3 concurrency levels). Instant failures (0.07s total).
- **Why skipped**: inferrs loads the model from local path (`$HF_MODEL`) successfully, but `/v1/models` reports `"Qwen/Qwen3-0.6B"` (HF repo name from model metadata). benchmark.py auto-detects this model ID and uses it in requests. inferrs doesn't match it to the already-loaded model and tries to re-download from HuggingFace, which fails (TLS error: `UnknownIssuer`). Attempted fix (passing `$HF_REPO` instead of `$HF_MODEL`) also fails due to same TLS issue. Root cause is an inferrs model-identity bug — the model ID it reports via `/v1/models` doesn't match what it accepts in request payloads when loaded from a local path.
- **Prior occurrences**: None in journals (first weekly-bench run). CLAUDE.md indicates inferrs worked at ~4.5 tok/s historically; likely broke during this week's `update_all.sh`.
- **Relevant logs**: `.frameworks/inferrs_server.log`, `results/Qwen3-0.6B/weekly_2026-04-11.log`
